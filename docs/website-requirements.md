# 個人網站需求重點

## 目標

- 首頁是文章索引，不是求職 Landing Page。
- 讀者先看到朱彥勳的公開文章與簡短介紹，再自行前往專案、經歷、關於或履歷。
- 網站只放經過整理且願意公開的內容；私人知識庫保持分離。

## 頁面

- Blog 首頁：彩色形象照、簡短自我介紹、文章搜尋／標籤與文章卡片。
- 文章內頁：Markdown／MDX、日期、閱讀時間、標籤與複製連結，不設留言。
- 專案：總覽卡片、搜尋／篩選與獨立詳情頁。
- 經歷：精簡時間條列，可展開閱讀詳細內容。
- 關於：個人背景、研究方向、技能與聯絡方式。
- 履歷：現有一頁式 HTML 與 PDF。

## 內容規則

- 介面支援繁體中文與英文切換；文章只呈現原始語言。
- 團隊專案必須區分整體成果與朱彥勳負責的部分。
- 不猜測專案資料、比賽結果或職責。
- 首頁暫時可放文章版型樣板，正式內容上線後移除樣板。
- 外部社群僅提供連結，不做自動發布。

## 維護與部署

- 專案與經歷繼續從 `_career_data` 同步，避免重複維護。
- 文章位於 `src/content/writing/`，草稿不進正式網站。
- 執行建置與桌面／手機檢查後才提交更新。
- GitHub Repository 保存原始碼；Cloudflare Pages 監聽 `main` 並輸出 `dist`。
- 正式網址為 `yanxunzhu.com`，`www` 轉址至根網域。
- 保留 canonical、sitemap、robots、Open Graph、結構化資料與 Web Analytics。
